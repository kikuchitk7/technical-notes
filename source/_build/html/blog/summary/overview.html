[第2回] AWS の機械学習サービスとは¶

はじめに¶

前回 は、本連載の背景についてご説明し、想定読者や目指す状態などを設定しました。 今回から AWS の機械学習サービスの具体的な内容を見ていきたいと思います。

なお、2019年12月1日〜6日にアメリカのラスベガスにて AWS の最大のイベントである AWS re:Invent 2019 が開催されました。 ここで多数の新サービスや既存機能のアップデートが発表されましたので、それを踏まえて整理していきます。

今回扱う内容と目標¶

  • 「AWS の機械学習サービスの種類、それぞれのサービスの概要や使い方」がわかり、他者に簡単に説明できる状態になる。

AWS の機械学習サービスとは¶

AWS の機械学習サービスについて順を追って整理していきます。

機械学習分野における AWS のミッション¶

AWS の機械学習サービスについて正しく知るためには、まず機械学習分野における AWS のミッションを知る必要があります。 それは、機械学習サービスの製品紹介ページにて宣言されている通り「すべての開発者に機械学習の恩恵を」です。

(出典：機械学習サービスの製品紹介ページ)

機械学習サービスの種類¶

機械学習サービスには大きく「AI サービス」、「ML サービス」、「ML フレームワークとインフラストラクチャ」の3種類があります。 上記のミッションを踏まえると、それぞれのサービスの特徴を把握しやすくなります。

AWS の機械学習サービス¶

機械学習サービス

特徴

用途

AI サービス

機械学習の深い知識なしで利用可能

機械学習アルゴリズムの実装をせずに利用したい人向け

ML サービス

機械学習のプロセス全体を効率化するマネージドサービス

独自に機械学習アルゴリズムを実装したい人向け

ML フレームワークとインフラストラクチャ

機械学習の環境を自在に構築して利用

独自に機械学習アルゴリズムを実装したい人向け

AI サービスは AWS によって機械学習モデルが提供される。 利用者はデータの用意だけをすれば良い。 メリット：手軽さ、デメリット：用途が限定される PoC やプロトタイプ作成などの利用も可能。 AWS re:Invent 2019 を経て、大きく分けて13種類になった。

ML サービスは少し乱暴にいうと、 Amazon SageMaker とそのサブ機能の集合。 AI サービスで要件を満たせない、独自に機械学習モデルを構築したい人向け。 AWS re:Invent 2019 を経て、 Amazon SageMaker は機能強化された。

ML フレームワークとインフラストラクチャは ML サービスとセットで考える。 図にあるものが ML サービスの基盤として提供されている。 Amazon 独自の機械学習の推論用チップ Inferentia を搭載した Inf1 インスタンスが発表された。

(出典：2019年12月12日開催 AWS re:Invent 2019 re:Cap | AI/ML の資料より)
