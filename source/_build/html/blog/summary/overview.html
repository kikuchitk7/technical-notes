[第2回] AWS の機械学習サービスとは¶

はじめに¶

前回 は、本連載の背景についてご説明し、想定読者や目指す状態などを設定しました。 今回から AWS の機械学習サービスの具体的な内容を見ていきたいと思います。

なお、2019年12月1日〜6日にアメリカのラスベガスにて AWS 社の最大のイベントである AWS re:Invent 2019 が開催されました。 ここで多数の新サービスや既存機能のアップデートが発表されましたので、それを踏まえて整理していきます。

扱う内容¶

  • 「AWS の機械学習サービスの種類、それぞれのサービスの概要や使い方」を整理する

目標¶

  • 「AWS の機械学習サービスの種類、それぞれのサービスの概要や使い方がわかる」という状態になる。

  • 同僚や上司に説明できる。

AWS の機械学習サービスとは¶

機械学習分野での AWS のミッション、コンセプト¶

  • 全ての人に機械学習を届ける

  • 3種類 (カテゴリ) を見るとそれがよくわかる。

機械学習サービスの種類¶

  • 大きくは3種類 (カテゴリ) ある。(図を re:Cap 資料などから持ってくる)

    • AI サービス

    • ML サービス

    • ML フレームワーク&インフラストラクチャ

機械学習サービスの選択の仕方¶

  • 仮説：種類がたくさんあってどれを使って良いのかわからないのでは？

  • それぞれのサービスの内訳を見る前に選択基準を整理する。

    • フローチャートか何かで選択基準を提案する

    • (※ 2回目はここで切れそう)

  • それぞれの特徴

    • 3つのサービスの内訳を見ていく

    • それぞれのサービスを一言で定義する

    • 詳細はサービスの紹介ページや開発者ガイドを見てもらう

      • サービスの紹介ページと Black Belt はリンクを紹介する

    • それぞれのユースケースを説明する。例えばこのような使い方をするなど。

    • (オプション) 情報収集の仕方を Tips 的に紹介する

まとめ¶

AWS の機械学習サービスの種類、それぞれのサービスの概要や使い方を整理することができたので、次回からは各サービスの具体的な実装について見ていく。という流れで。

著者紹介¶

菊地 貴彰 (KIKUCHI Takaaki)

株式会社 NTT データ システム技術本部 デジタル技術部 Agile Professional Center

大学・大学院では、機械学習を専攻。 ベイズ的枠組みを用いて、複数の遺伝子のデータから遺伝子どうしの相互作用ネットワークの推定に関する研究を行った。

株式会社NTTデータに入社後は、法人や金融のシステム開発のシステム基盤担当としてキャリアを積み、 現在はデジタル技術や Agile 開発を専門に扱う組織でシステム開発全般を担当する。 2019 APN AWS Top Engineers に選出。
